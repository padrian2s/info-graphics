1. Page Structure

Every page in the Info Graphics Hub should follow a consistent structure for maintainability and user experience.

Required Elements

  • DOCTYPE and HTML5: All pages must use <!DOCTYPE html>
  • Viewport meta tag: Essential for responsive design
  • Title: Descriptive title with site suffix
  • Navigation: Link back to the hub
  • Main content area: Semantic <main> element
Basic Page Template
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title | Info Graphics Hub</title>
    <style>
        /* Include shared CSS variables */
        :root {
            --primary: #2563eb;
            --bg: #f8fafc;
            --card-bg: #ffffff;
            --text: #1e293b;
            --text-muted: #64748b;
            --border: #e2e8f0;
            --radius: 8px;
        }
        /* Page-specific styles... */
    </style>
</head>
<body>
    <nav class="nav">
        <a href="index.html">‚Üê Back to Hub</a>
    </nav>

    <main class="container">
        <!-- Page content here -->
    </main>

</html>

<meta name="category" content="Reports" />
<meta name="tags" content="tag1, tag2, tag3" />

3. HTML Standards

Semantic Markup

Use semantic HTML5 elements to improve accessibility and SEO:

  • <header> ‚Äì Page or section headers
  • <nav> ‚Äì Navigation blocks
  • <main> ‚Äì Primary page content
  • <section> ‚Äì Thematic groupings
  • <article> ‚Äì Self-contained content
  • <aside> ‚Äì Supplementary content
  • <footer> ‚Äì Page or section footers

Heading Hierarchy

Maintain proper heading structure for accessibility:

  • One <h1> per page (the main title)
  • Use headings in order: h1 ‚Üí h2 ‚Üí h3
  • Don't skip levels (e.g., h2 ‚Üí h4)
üí° Tip Use heading elements for structure, not for styling. Use CSS classes to control visual appearance.

All pages should use the shared CSS custom properties (variables) for consistency:

    --text: #1e293b;           /* Primary text */

Include dark mode variants using the prefers-color-scheme media query:

@media (prefers-color-scheme: dark) {
    :root {
        --bg: #0f172a;
        --card-bg: #1e293b;
        --text: #f1f5f9;
        --text-muted: #94a3b8;

    }
}

CSS Organization

  1. CSS variables (`:root`)
  2. Reset/base styles
  3. Layout (containers, grids)
  4. Component styles (cards, buttons)
  5. Utility classes
  6. Media queries (mobile-first or desktop-first, be consistent)

5. Accessibility Requirements

‚ö†Ô∏è Important Accessibility is not optional. All pages must meet WCAG 2.1 Level AA standards.

Required Practices

  • Language attribute: Always set lang="en" on the <html> element
  • Alt text: All <img> elements must have descriptive alt attributes
  • Form labels: All form inputs must have associated <label> elements
  • Focus indicators: Never remove focus outlines without providing alternatives
  • Color contrast: Text must have 4.5:1 contrast ratio minimum
  • Keyboard navigation: All interactive elements must be keyboard accessible

ARIA Guidelines

  • Use native HTML elements before ARIA (e.g., <button> over role="button")
  • Add aria-label for icon-only buttons
  • Use aria-live for dynamic content updates

8. Responsive Design

Breakpoints

Name Width Target Devices
Mobile < 640px Phones
Tablet 640px ‚Äì 1024px Tablets, small laptops
Desktop > 1024px Laptops, desktops

Best Practices

  • Use relative units (rem, em, %) over fixed pixels
  • Set max-width on containers, not fixed widths
  • Use CSS Grid or Flexbox for layouts
  • Test on real devices when possible
  • Use min-width in media queries for mobile-first approach
/* Mobile-first approach */
.grid {
    display: grid;
    grid-template-columns: 1fr;
    gap: 1rem;
}

@media (min-width: 640px) {
    .grid {
        grid-template-columns: repeat(2, 1fr);
    }
}

@media (min-width: 1024px) {
    .grid {
        grid-template-columns: repeat(3, 1fr);
    }
}

9. Performance Best Practices

General Guidelines

  • Keep pages lightweight: Target under 100KB total (HTML + CSS + JS)
  • Minimize dependencies: Avoid external libraries when vanilla JS suffices
  • Inline critical CSS: Include essential styles in the <style> tag
  • Lazy load images: Use loading="lazy" on images below the fold
  • Optimize images: Use WebP format, appropriate dimensions

JavaScript Guidelines

  • Place scripts at the end of <body> or use defer attribute
  • Use event delegation for better performance
  • Debounce search input and scroll handlers
  • Avoid blocking the main thread with heavy computations
üí° Performance Testing Use Chrome DevTools Lighthouse to audit pages. Aim for 90+ scores in Performance and Accessibility.
